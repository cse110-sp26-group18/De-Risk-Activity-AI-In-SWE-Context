# AI Policy

## Purpose
This document outlines how AI tools will be used in both the planning and execution phases of the project to maximize efficiency while maintaining quality and control.

---

## AI Usage Strategy

### Planning Phase
For planning purposes, we will use Claude Opus to generate cleaner, more structured, and well-thought-out code and ideas. This model is preferred for higher-quality outputs when designing systems, features, and workflows.

### Execution Phase
For execution tasks, we will use Claude Sonnet to reduce token usage and improve efficiency. This helps keep costs lower while still maintaining reasonable output quality.

---

## Code Generation Guidelines
We will rely on AI to generate code where appropriate, especially for repetitive tasks. However, we will prioritize understanding and reviewing all generated code before integrating it into the project.

---

## Documentation Approach
While AI may assist with generating drafts, the team will primarily handle documentation manually to ensure clarity, accuracy, and alignment with project goals.

---

## AI Limitations Policy
If an AI model is unable to successfully complete a task after two well-constructed prompts, we will either:
- Implement the solution manually, or  
- Reevaluate and potentially scrap the feature or approach  

---

## Prompt Engineering Guidelines
Prompts will be initially written by hand to ensure intent and clarity. Before sending them to the main AI model, prompts may be refined using an LLM to improve effectiveness and output quality.
